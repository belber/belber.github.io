# 前端学习路线 — 极客时间 × 微信读书

> 从管理视角，串联课程与书籍，按阶段系统入门  
> 极客时间 12 门课程 · 微信读书 13 本经典

---

## 第一阶段：地基 — 理解前端本质

> 核心是理解"前端到底是什么、浏览器怎么工作、网络怎么通信"

### 🎓 极客时间课程

| 课程 | 讲师 | 讲数 | 推荐理由 |
|------|------|------|----------|
| **重学前端** | 程劭非（winter） | 62 讲 | **管理者首选**。不讲 API，讲"前端知识体系"本身。帮你在脑中建立完整的认知框架。 |
| **浏览器工作原理与实践** | 李兵 | 46 讲 | **理解运行环境**。进程线程、渲染流水线、事件循环——所有前端框架要解决的根本问题。 |
| **透视 HTTP 协议** | 罗剑锋（Chrono） | 48 讲 | **性能优化核心**。比"趣谈网络"更聚焦前端场景，HTTP 是前端与服务器的桥梁。 |

### 📖 微信读书

| 书名 | 作者 | 评分 | 推荐理由 |
|------|------|------|----------|
| **图解HTTP** | 上野宣 | ⭐ 8.0 | 图文并茂，2 天读完。快速理解 HTTP 全貌的最佳入门书。 |
| **Web性能权威指南** | Ilya Grigorik | ⭐ 8.6 | 从 TCP、TLS 到 HTTP/2，打通前端性能优化的底层认知。 |
| **程序员的自我修养** | 俞甲子 | ⭐ 8.5 | 编译、链接、库——所有端侧开发者的通用内功。 |

## 第二阶段：语言 — 掌握核心工具

> JavaScript 是你管理所有 H5 团队的语言基础，TypeScript 是现代前端的标配

### 🎓 极客时间课程

| 课程 | 讲师 | 讲数 | 推荐理由 |
|------|------|------|----------|
| **JavaScript 核心原理解析** | 周爱民 | 28 讲 | **短小精悍**。不讲语法，讲语言设计思想。理解 JS 的设计选择——这对理解 React/Vue 至关重要。 |
| **TypeScript 开发实战** | 梁宵 | 47 讲 | **一通百通**：TS 是 JS 的超集，ArkTS 又是 TS 的衍生。理解 TS 就理解了为什么 Angular/ArkUI 走类型约束路线。 |

### 📖 微信读书

| 书名 | 作者 | 评分 | 推荐理由 |
|------|------|------|----------|
| **JavaScript高级程序设计（第4版）** | Matt Frisbie | ⭐ 9.0+ | 前端从业者必读"**红宝书**"，系统全面。作为管理者，读第 1~5 章建立框架即可。 |
| **你不知道的 JavaScript（上卷）** | Kyle Simpson | ⭐ 9.2 | **管理者必读**。作用域、闭包、this 绑定——JS 最难也是最核心的部分。评估团队技术水平的重要标尺。 |
| **你不知道的 JavaScript（中卷）** | Kyle Simpson | ⭐ 9.0 | 类型、异步、性能。看完对 JS 的异步机制有质的理解。 |
| **ECMAScript 6 入门** | 阮一峰 | ⭐ 8.5 | ES6+ 语法速查圣经，遇到不熟的语法翻一翻。 |

## 第三阶段：框架 — 理解声明式 UI 革命

> 这个阶段要把 React、Vue、SwiftUI、Compose、ArkUI 的**共性**看出来——它们都是"声明式 UI + 响应式状态管理"

### 🎓 极客时间课程

| 课程 | 讲师 | 讲数 | 推荐理由 |
|------|------|------|----------|
| **React 实战进阶 45 讲** | 王沛 | 46 讲 | **理解声明式 UI 的起点**。React 的"单向数据流 + 组件化"影响了所有现代 UI 框架（包括 SwiftUI 和 Compose）。 |
| **玩转 Vue 3 全家桶** | 大圣 | 44 讲 | 大圣的特色是从"**框架发展史讲知识体系**"——刚好适合需要建立全局观的你。 |

### 📖 微信读书

| 书名 | 作者 | 评分 | 推荐理由 |
|------|------|------|----------|
| **CSS揭秘** | Lea Verou | ⭐ 9.4 | **前端界评分最高的书之一**。不讲基础，讲"CSS 还能这样用"——帮你建立视觉表现力的认知边界。 |
| **Vue.js 设计与实现** | 霍春阳 | ⭐ 9.0+ | **最好的 Vue 3 原理书**。响应式系统、虚拟 DOM、编译器全部讲透。 |
| **深入 React 技术栈** | 陈屹 | ⭐ 7.8 | React 核心原理 + 生态全景（Redux、React Native）。适合理解 React 全貌。 |
| **React 设计原理** | 卡辛 | ⭐ 8.5 | 从设计者角度讲 React 为什么这么设计——更适合**管理者的视角**。 |

> **核心建议**：Vue.js 设计与实现 / React 设计原理 选一本深入即可。关键是理解了"声明式 UI + Virtual DOM + 响应式状态"的通用模型，你就自然理解 SwiftUI / Compose / ArkUI 的设计。

## 建议优先级（作为管理者）

### 第一优先 · 建立认知框架

- **课程**：重学前端 + 浏览器原理 + 透视 HTTP
- **书籍**：图解HTTP + Web性能权威指南

### 第二优先 · 理解语言本质

- **课程**：JS 核心原理解析 + TypeScript 实战
- **书籍**：你不知道的 JS（上卷）+ CSS揭秘

### 第三优先 · 理解声明式 UI

- **课程**：React 进阶（或 Vue 3）+ Flutter 核心
- **书籍**：Vue.js 设计与实现（或 React 设计原理）+ 前端架构

### 第四优先 · 理解各端生态

- **课程**：Kotlin 上手（选前 20 讲）+ Swift（选前 20 讲）+ HarmonyOS 入门
- **书籍**：TypeScript 编程

### 第五优先 · 展望未来

- **课程**：跟月影学前端智能体开发

---

> **核心建议**：不要同时看太多课。按阶段走，每个阶段只深入 1-2 门。重点是**对比着看**——比如学 React 时想想 Vue 怎么做、学 SwiftUI 时想想 React 怎么做，这样才能把知识串联起来。
